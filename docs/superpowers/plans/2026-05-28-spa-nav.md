# SPA Navigation (Build Step 1.5) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** When the user navigates between Scalable assets without a full page reload, the Badge re-renders with the new ISIN (or disappears if the new URL is not a security page). No more hard-reload requirement.

**Architecture:**
- Scalable's React SPA changes the URL with `history.pushState`/`replaceState` — these do **not** fire `popstate`. The standard fix is to monkey-patch both methods to emit a custom event after call, and listen for `popstate` (covers back/forward) plus the custom event (covers programmatic navigation).
- A pure `observeIsin(window, onChange)` function in `src/lib/` encapsulates the observer + cleanup; it emits the current ISIN (or `null`) on every detected URL change and on initial call.
- The content script holds a tiny `currentIsin` state and reconciles: same → noop, new ISIN → render `<Badge>`, null → unmount host.

**Tech Stack:** Vanilla DOM + Preact `render`/`render(null, host)` for unmount. jsdom for unit tests.

### Design

```ts
export type IsinListener = (isin: string | null) => void;
export function observeIsin(win: Window, onChange: IsinListener): () => void;
```

Behaviour:
- On call, immediately emits `parseIsinFromUrl(win.location.href)`.
- Monkey-patches `win.history.pushState` and `win.history.replaceState`: after the original call runs, dispatch a `CustomEvent("ape-intel:locationchange")` on `win`.
- Adds `win.addEventListener("popstate", handler)` and `win.addEventListener("ape-intel:locationchange", handler)`.
- The handler computes the new ISIN and, if different from the last emitted value, calls `onChange(isin)`. (De-duping prevents redundant renders when Scalable re-pushes the same URL.)
- The returned cleanup function: restores the two original history methods and removes both listeners.

Edge cases the tests must cover:
- Initial emit happens synchronously on call.
- `pushState` to a security URL with a new ISIN → listener fires with new ISIN.
- `replaceState` to a non-security URL → listener fires with `null`.
- `popstate` (simulated via dispatching the event) → listener fires.
- Same-URL `pushState` → listener does NOT fire (dedup).
- Cleanup: after cleanup, history methods are restored (`win.history.pushState === original`) and further `pushState` does not call the listener.

- [ ] **Step 3: Implement `src/lib/url-observer.ts`**

```ts
import { parseIsinFromUrl } from "./isin";

export type IsinListener = (isin: string | null) => void;

const LOCATION_CHANGE_EVENT = "ape-intel:locationchange";

export function observeIsin(win: Window, onChange: IsinListener): () => void {
  let lastUrl = win.location.href;
  let lastIsin = parseIsinFromUrl(lastUrl);

  const originalPush = win.history.pushState;
  const originalReplace = win.history.replaceState;

  const wrap = (
    fn: typeof originalPush,
  ): typeof originalPush =>
    function patched(this: History, ...args) {
      const result = fn.apply(this, args);
      win.dispatchEvent(new Event(LOCATION_CHANGE_EVENT));
      return result;
    } as typeof originalPush;

  win.history.pushState = wrap(originalPush);
  win.history.replaceState = wrap(originalReplace);

  const handler = (): void => {
    const url = win.location.href;
    if (url === lastUrl) return;
    lastUrl = url;
    const isin = parseIsinFromUrl(url);
    if (isin === lastIsin) return;
    lastIsin = isin;
    onChange(isin);
  };

  win.addEventListener("popstate", handler);
  win.addEventListener(LOCATION_CHANGE_EVENT, handler);

  onChange(lastIsin);

  return () => {
    win.history.pushState = originalPush;
    win.history.replaceState = originalReplace;
    win.removeEventListener("popstate", handler);
    win.removeEventListener(LOCATION_CHANGE_EVENT, handler);
  };
}
```

## Task 2: Wire observer into content script

**Files:**
- Modify: `src/content/index.tsx`

The script becomes reactive: it owns a single host div, calls `observeIsin`, and reconciles by either rendering `<Badge>` with the new ISIN or unmounting via `render(null, host)`.

### Steps

- [ ] **Step 1: Replace `src/content/index.tsx` with the reactive version**

```tsx
import { render } from "preact";
import { Badge } from "./Badge";
import { observeIsin } from "../lib/url-observer";

const HOST_ID = "ape-intel-host";

function ensureHost(): HTMLElement {
  const existing = document.getElementById(HOST_ID);
  if (existing) return existing;
  const host = document.createElement("div");
  host.id = HOST_ID;
  document.body.appendChild(host);
  return host;
}

observeIsin(window, (isin) => {
  if (isin) {
    render(<Badge isin={isin} />, ensureHost());
  } else {
    const host = document.getElementById(HOST_ID);
    if (host) render(null, host);
  }
});
```

Notes:
- `parseIsinFromUrl` is no longer imported here — `observeIsin` already calls it.
- `render(null, host)` is Preact's documented unmount; we keep the host element in place so re-mounts are cheap and the badge does not flicker if the user toggles between assets quickly.
- No cleanup is wired up; content scripts have the same lifecycle as the page, and there is no second observer to dispose. If we later add hot-reload tooling, capture the returned cleanup.
